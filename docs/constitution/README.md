# Novara core / constitution docs

Novara の「憲法レイヤ」をまとめるディレクトリ。

- コードは置かない
- 数学証明もしない
- 「何を許さないか」「誰がいつ変えていいか」だけを書く

2026 年までは **v0.1 = 方針メモ**。  
実装は Novara Evidence OS 側で動かしつつ、ここにはその「理由」と「境界条件」だけ残す。

---

## どの順番で読むか

1. `novara-constitution-index-v0.1-ja.md`  
   Novara 憲法レイヤの目次・用語・スコープ。

2. `novara-anti-capture-v0.1-ja.md`  
   資本・国家・単一ベンダーに乗っ取らせないための原則。

3. `novara-anti-abuse-v0.1-ja.md`  
   権威主義的な監視 OS に転ぶのを防ぐための境界条件。

4. `novara-capital-policy-v0.1-ja.md`  
   財団 / 営利会社 / 投資家の役割と、譲ってはいけない線。

5. `novara-meta-asi-governance-v0.1-ja.md`  
   ASI が「Novara のルール自体」を最適化対象にしたときの防御線。

---

## 2026 までの使い方

- ここに書いてあるのは **「このラインを踏んだら負け」** という禁止リストが中心。
- 仕様詳細・API・証拠形式は `Novara` リポジトリ側（Evidence OS）に書く。
- バージョン番号はゆっくり動かす：v0.1 のままでも、追記だけしてよい。

将来、本当に裁判・保険・標準化で使う段階になったら：

- v1.x 系として英語版を追加
- 条文ごとに「根拠となる判例・規格・運用実績」をリンクで紐付ける
