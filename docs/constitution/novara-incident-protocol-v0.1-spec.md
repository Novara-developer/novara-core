# インシデント・プロトコル v0.1 ドラフト（v0.1 リセット）

このファイルは、旧 v0.1 インシデント・プロトコルを **廃棄して**、
「事故が起きたときに何をどう記録するか」をゼロベースで書き直すためのプレースホルダ。

旧版は Git の履歴にだけ残す。  
main ブランチ上では、ここから新しいプロトコルだけを見る。

---

## 目的

- 事故・障害・ハラスメント・不正など **「人にダメージが出るイベント」** が起きたときに、
  - いつ
  - 誰が
  - 何を
  - どの形式で
  Evidence Bundle に刻むかを固定する

---

## ここに定義するべきもの（アウトライン）

1. タイムライン
   - T0: 発見／通報
   - T+24h: 一次封じ込め ＋ 暫定レポート
   - T+7d: 詳細レポート（Evidence Bundle 完成）
   - T+30d: SLO-Bound Remedy の実行確認

2. 必須ログ
   - 事故前の AAL スナップショット
   - 事故発生時のコンテキスト
   - オペレーター／AI／ポリシーのバージョン
   - すべての「判断」と「拒否」の記録

3. 公開／非公開の境界
   - 被害者・裁判所・保険会社に開示する範囲
   - 市民／メディアに公開する要約
   - 匿名化・マスキングのルール

4. Novara Cases との接続
   - `novara-incidents` リポで公開する時の最低条件
   - Evidence Bundle と Human-Readable Proof のセット構成
