# Novara 憲法レイヤ index v0.1

目的
- novara-core の中で「どのファイルが正式なルールか」を一枚で見えるようにする
- 将来ファイルを増減しても、この index を見れば全体像が分かるようにする

この文書で決めること
- docs/constitution の中で「生きている」文書の一覧
- 各文書がおおざっぱに何を決めているか
- 憲法レイヤのファイル追加と削除の運用ルール

チェックリスト
- [ ] 下の一覧に無いファイルを「公式な憲法文書」と呼ばない
- [ ] 新しい文書を追加したら、必ずここに一行説明を足す
- [ ] 文書を消したら、ここからも消す

文書一覧 v0.1

1. novara-principles-v0.1-ja.md  
   Novara の上位原則を十個にまとめたもの。ほか全部より優先順位が高い。

2. novara-evidence-sovereignty-v0.1-ja.md  
   何をどこまでログにするか、誰が開示を決めるか、その境界線だけを決める。

3. novara-meta-asi-guard-v0.1-ja.md  
   ASI が Novara のルール自体を最適化対象にした時に、どこから拒否するかの枠組み。

4. novara-standardisation-strategy-outline-v0.1-ja.md  
   どの標準化フォーラムにどう入り、何を取りに行くかの骨組み。

5. 90-glossary-ja.md  
   用語の最小辞書。外部の人間が迷子にならないための入口。

運用ルール v0.1

- 新しい憲法系ファイルを作る時は、先にファイルを追加し、そのあとでこの index に一行説明を書く
- そのうち書きたいメモは docs/archive 以下に置き、この一覧には入れない
- 年度ごとの大きな改訂をしたら、コミットメッセージに「core-constitution-v0.x」などのタグを入れる
