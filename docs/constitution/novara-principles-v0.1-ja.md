# Novara core 原則 v0.1

目的
- 「これを壊したら Novara を名乗るな」というラインを十個に絞って言語化する
- 実装の細かい話は Novara 実装側に追い出し、ここには方向性だけを残す

この文書で決めること
- Novara の上位原則十個
- どの原則も一行で意味が分かるレベルに圧縮すること
- 下位文書がこの原則と矛盾した場合は、下位文書を修正するという優先順位

チェックリスト
- [ ] 原則は十個より増やさない
- [ ] どの原則も二行以内で読める
- [ ] 技術者と法務と保険屋が読んでも同じ方向に解釈できる

原則 v0.1

1. Evidence first  
   新しい機能を出す前に、証拠形式と検証手順を先に決める。

2. Evidence sovereignty  
   何が起きたかの最終判断は、ダッシュボードではなく evidence bundle の中身で行う。

3. Zero trust verification  
   ベンダーやクラウドを信用しなくても、公開された素材だけで検算できる。

4. Determinism  
   同じ入力と環境からは、同じ evidence bundle が出る。違った場合は必ず差分理由を残す。

5. Tamper evidence  
   ログや設定は、消せないか、消した痕跡が暗号的に残るようにする。

6. Attribution ready  
   各 harm を、モデル、ポリシー、時間帯、主体まで遡れるように設計する。

7. SLO bound remedy  
   一定以上の harm には、事前に決めた式で自動的に補償額が決まるようにする。

8. Human readable proof  
   被害者と市民には、ハッシュ列だけでなく、自然言語の説明が渡るようにする。

9. Multi anchor neutrality  
   アンカーは、政治ブロックもオペレーターも分散させ、一箇所に依存しない。

10. Civic time horizon  
    特定企業や政権ではなく、二〇四〇から二〇六〇に生きる市民に向けて設計する。
