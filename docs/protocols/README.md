# Novara プロトコル群 v0.1

このディレクトリは「コードではなく手順」を固定する場所。

- 事故が起きたときにどう動くか
- 被害者や一般市民にどう説明するか
- コア条文をどう変更していいか、どこまでが権限か

など、運用・説明・改訂プロセスだけをここに集約する。

実装や API は Novara リポジトリ側の責任範囲。  
ここは「こう動かなかったら Novara を名乗るな」という最低ラインを文章で定義する。

v0.1 では次の三つだけを用意する。

- novara-incident-protocol-outline-v0.1-ja.md  
  事故発生から 30 日後までの最低限フローの叩き台

- novara-human-readable-proof-outline-v0.1-ja.md  
  一般人向けの「人間が読める証拠説明」のテンプレ案

- novara-constitution-change-process-outline-v0.1-ja.md  
  コア条文を変える時のプロセスと、やってはいけない変更ライン

どれもまだ「outline」。  
将来、実際の PoC や事件から学んだ内容を踏まえて v0.2, v0.3... と更新していく。
